Q-1: Difference between localStorage and sessionStorage

Ans:

localStorage and sessionStorage are almost identical and have the same API. The difference is that with sessionStorage , the data is persisted only until the window or tab is closed. With localStorage , the data is persisted until the user manually clears the browser cache or until your web app clears the data.

Question-3 : How event loop work in javaScript

Ans:

We know that javascript is Synchronous , But when it runs on Browser ,browser api provide asynchronous behaviour to javaScript with the help of Event Loop .

Event loop monitor the call Stack and the Callback Queue . If the Call stack is empty , the Event loop will take the first event from the queue and will push it to the call Stack , Which effectively runs it .

Question-4 : How can we get undefined ?

Ans:

we can get undefined in many ways, examples are given below:

1. A variable or an object has an undefined value when no value is assigned before using it .

2. if we try to access a variable that doesn't exist will give us undefined

3. A function that doesn't return anything will give you undefined if you try to use it .

4.A property that doesn't exist , if you want to access it , then you will get undefined.

5.An array element that doesnt exist , if you want to access it you will get undefined.
